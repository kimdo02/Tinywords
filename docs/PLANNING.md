# TinyWords 기획서

> 영어 유치원에 다니는 아이(5~7세)가 **잘 안 외워지는 단어 / 시험에서 틀린 단어**를
> 등록해두고, **게임 형식으로 재미있게 반복 학습**하고 **간단한 테스트**로 확인하는 PWA.

---

## 1. 제품 방향

### 문제
- 유치원에서 배운 영어 단어 중 아이가 특히 못 외우거나 시험에서 틀리는 단어가 있다.
- 반복 학습이 필요하지만, 단순 암기는 아이가 지루해하고 안 한다.

### 해결
- 부모가 "약한 단어"만 골라 등록 → 아이가 **놀이(게임)로 반복** → **테스트로 확인**.
- 오답이 많은 단어를 자동으로 더 자주 노출해 효율적으로 약점 보완.

### 성공 기준 (초기 지표)
- 아이가 스스로 앱을 열어 하루 1세션 이상 플레이한다.
- 등록된 단어의 `mastered` 비율이 시간에 따라 올라간다.
- 부모가 단어 등록을 1분 내에 부담 없이 끝낸다.

---

## 2. 사용자 & 역할

| 역할 | 하는 일 | 특징 |
|------|---------|------|
| 부모 | 로그인, 단어 등록/관리, 진도 확인 | 모바일 우선, 빠른 입력 필요 |
| 아이(5~7세) | 게임 학습, 테스트 | 읽기 미숙 → 음성/아이콘/이미지 중심 |

- **인증/동기화 모델**: 부모가 이메일로 로그인 → 계정 안에 **여러 아이 프로필** 저장
  → 기기에서 프로필 선택. 로그인 세션이 기기 간 동기화 단위가 된다.

---

## 3. 기술 개요

- **프론트엔드**: Next.js (App Router) + TypeScript + Tailwind CSS
- **백엔드**: Supabase (Postgres + Storage + Auth)
- **발음(TTS)**: MVP는 브라우저 Web Speech API + **원어민 음성(en-US/en-GB) 선택 & 속도(rate)·톤(pitch) 조절**.
  일관된 원어민 발음이 중요하면 이후 클라우드 TTS로 업그레이드(§10 참고).
- **말하기 인식**: 발음 따라 말하기 게임용 Web Speech Recognition (판정 관대하게).
- **형태**: 설치형 PWA / **배포**: Vercel

## 4. 데이터 모델 (초안)

```
profiles (아이 프로필)
  id           uuid pk
  owner        uuid  -- auth.uid() (부모)
  name         text
  avatar       text  -- 아이콘/색상 선택값
  created_at   timestamptz

words (단어)
  id             uuid pk
  profile_id     uuid fk -> profiles
  term           text   -- 영어 단어
  meaning        text   -- 대표 한글 뜻 (학습·채점 기준)
  alt_meanings   text[]? -- 참고용 추가 뜻 (여러 뜻일 때, 선택)
  example        text?  -- 예문 (문장 빈칸 게임에 사용)
  image_url      text?  -- Supabase Storage 경로 (없어도 됨)
  emoji          text?  -- 이미지 없을 때 대체 비주얼 (선택)
  category       text?  -- 동물/음식 등 태그
  difficulty     int    -- 1~3 (선택)
  wrong_count    int    default 0
  correct_count  int    default 0
  streak         int    default 0  -- 연속 정답 수 (mastered 승격 판정용)
  status         text   -- 'learning' | 'mastered'
  last_reviewed_at timestamptz?
  created_at     timestamptz

test_results (테스트 결과)
  id            uuid pk
  profile_id    uuid fk -> profiles
  taken_at      timestamptz
  total         int
  correct       int
  wrong_word_ids uuid[]

account_settings (부모 계정 설정)
  owner            uuid pk  -- auth.uid()
  pin_hash         text?    -- 부모 모드 잠금 PIN(해시 저장)
  tts_voice        text?    -- 선택한 원어민 음성
  tts_rate         numeric default 1.0  -- 발음 속도
  tts_pitch        numeric default 1.0  -- 발음 톤
  default_test_count int   default 10   -- 테스트 기본 문항 수(부모가 5~10 선택)
  session_size     int     default 8    -- 게임 한 세션 단어 수
```

- **뜻이 여러 개인 단어**: `meaning`(대표 뜻)만 채점/학습 기준으로 사용,
  나머지는 `alt_meanings`에 참고용으로만 저장·표시.
- **이미지가 어려운 단어**: `image_url`은 선택. 대신 `emoji`로 대체 비주얼 지정 가능하고,
  둘 다 없으면 뜻+예문+TTS로 학습. **그림이 필요한 게임은 image/emoji가 있는 단어만** 대상.
- **RLS**: 모든 테이블은 `auth.uid()`가 프로필 소유자(또는 계정 주인)일 때만 접근 가능.
- **숙련도 전이**: `streak`가 **연속 3회** 정답이면 `learning → mastered`,
  오답 시 `streak=0` 및 `learning`으로 복귀. 임계치(3)는 상수로 관리해 조정 가능하게.

## 5. 화면 구성 (라우트)

| 경로 | 화면 | 대상 |
|------|------|------|
| `/login` | 부모 이메일 로그인 | 부모 |
| `/profiles` | 아이 프로필 생성/선택 | 부모/공용 |
| `/` (home) | 아이 홈 — 오늘의 학습/테스트 큰 버튼 | 아이 |
| `/study` | 게임 선택 → 플레이 | 아이 |
| `/test` | 테스트 진행 → 결과 | 아이 |
| `/words` | 단어 목록/등록/수정 (부모 모드, **PIN 잠금**) | 부모 |
| `/progress` | 진도 + **자주 틀리는 단어 통계** (부모 모드, PIN 잠금) | 부모 |

- 아이 화면과 부모 화면의 톤 분리(아이=놀이, 부모=간결/효율).
- **부모 모드(`/words`, `/progress`) 진입 시 PIN 입력** — 아이가 관리 화면에 못 들어가게 보호.
  최초 진입 시 PIN 설정, 이후 진입마다 확인(세션 동안 유지 가능).

## 6. 핵심 시나리오

### 시나리오 A — 부모가 틀린 단어를 등록
1. 부모가 로그인 후 아이 프로필 선택.
2. 부모 모드 진입 시 **PIN 입력** → `/words`에서 "단어 추가".
3. 영어 단어 + 대표 뜻 입력. 뜻이 여러 개면 추가 뜻(선택), 예문(선택) 입력.
4. (선택) 이미지 업로드 또는 이모지 선택, 카테고리 지정 후 저장.
5. 새 단어는 `status: learning`, `streak: 0`으로 학습 풀에 들어간다.

### 시나리오 B — 아이가 게임으로 학습
1. 아이가 홈에서 "공부하기"의 큰 버튼을 누른다.
2. 시스템이 `mastered`가 아닌 단어 중 `wrong_count` 높은 순으로 5~10개 선정.
3. 게임(플래시카드/뜻 맞히기/듣고 고르기/카드 매칭) 플레이.
4. 정답 → 별·사운드·애니메이션 보상 / 오답 → 부드럽게 정답 다시 보여주고 재시도.
5. 연속 정답 임계치 도달 단어는 `mastered`로 승격. 세션 끝에 획득한 별 요약.

### 시나리오 C — 테스트로 확인
1. 아이가 "시험 보기" 버튼을 누른다.
2. **문항 수 N**은 부모 모드에서 정한 값(5~10, 기본 10). 등록 단어가 N보다 적으면
   있는 만큼만 출제. 등록 단어 중 N개를 객관식/듣고 고르기로 출제 → 즉시 채점.
3. 결과를 `test_results`에 저장하고 **틀린 단어의 `wrong_count`를 증가**.
4. 결과 화면: 점수 + 틀린 단어 목록 + "이 단어 다시 연습" 버튼.
5. 부모는 `/progress`에서 최근 테스트 결과와 **자주 틀리는 단어 통계**를 확인.

### 시나리오 D — 부모가 통계로 약점 파악
1. 부모 모드(PIN) → `/progress` 진입.
2. `wrong_count` 상위 단어(자주 틀리는 단어) TOP 리스트, 최근 테스트 점수 추이,
   `mastered`/`learning` 비율을 요약해 보여준다.
3. 각 단어에서 바로 "이 단어만 연습" 세션을 시작할 수 있다.

## 7. 게임 상세

### 텍스트/오디오 기반 (모든 단어 사용 가능)
1. **플래시카드**: 앞면=영어 단어(+TTS 재생), 뒷면=한글 뜻/이미지. 넘기며 익힘.
2. **뜻 맞히기(객관식)**: 단어를 보고 4지선다 한글 뜻 선택.
3. **듣고 고르기**: TTS 발음을 듣고 맞는 단어/그림 선택.
4. **카드 매칭**: 단어 카드와 뜻(또는 이미지) 카드 짝 맞추기.
5. **스펠링 빈칸 채우기**: 단어의 일부 알파벳을 빈칸으로 두고 맞는 알파벳 선택.
   예: 🍎 + `Ap_le` → 정답 `p`. (TTS로 발음 힌트 제공)
6. **발음 따라 말하기(말하기 연습)**: 단어를 보여주고 아이가 마이크로 발음 →
   Web Speech Recognition으로 판정. **판정은 관대하게**, 실패해도 벌 없이 "잘했어!" 후 넘어감.

### 그림 기반 (image 또는 emoji 있는 단어만 대상)
7. **그림 보고 단어 맞히기**: 이미지/이모지를 보여주고 맞는 영어 단어 선택(또는 스펠링 입력).
8. **문장 빈칸 채우기**: 그림 + 그것을 설명하는 예문을 보여주고, 문장의 네모 칸에
   들어갈 알맞은 단어를 보기 중에서 선택. 예: 🍎 + "I eat an ___." → 정답 `apple`.

> - 오답 유도 보기는 같은 카테고리/난이도의 다른 단어에서 뽑아 난이도 조절.
> - 그림 기반 게임(7,8)은 image/emoji가 없는 단어는 자동 제외.
> - 문장 빈칸 게임(8)은 `example`(예문)이 있는 단어만 대상.

## 8. 게임화 & 아이 UX 원칙

- 큰 터치 타깃, 밝은 색, 최소 텍스트, 아이콘/이미지 중심.
- 영어 단어는 항상 TTS로 들려주기 버튼 제공.
- 정답은 즉각 칭찬/보상, 오답은 벌 없이 재시도.
- 별·스티커·진척 바로 성취를 시각화. 한 판은 1~3분으로 짧게.

## 9. 개발 로드맵 (단계)

- **1단계 — 기반**: 프로젝트 세팅(Next.js+TS+Tailwind), Supabase 연동, PWA 기본,
  부모 로그인 + 아이 프로필 + **부모 모드 PIN**.
- **2단계 — 단어 관리**: `/words` CRUD, 대표뜻/추가뜻/예문, 이미지 업로드·이모지, 카테고리.
- **3단계 — 학습 게임(기본)**: 플래시카드 → 뜻 맞히기 → 듣고 고르기 → 카드 매칭.
  TTS(원어민 음성/속도/톤 설정), 단어 선정 로직, `streak` 기반 숙련도 전이(3회).
- **4단계 — 학습 게임(확장)**: 스펠링 빈칸, 그림 보고 단어 맞히기, 문장 빈칸 채우기,
  발음 따라 말하기(음성인식).
- **5단계 — 테스트**: 테스트 출제/채점, 결과 저장, 오답 반영.
- **6단계 — 진도/통계 & 다듬기**: `/progress`(자주 틀리는 단어 통계), 사운드/애니메이션,
  PWA 설치 경험 개선.
- **7단계 — (이후) 확장**: 오프라인 강화, 클라우드 TTS, 프로필별 테마 등.

## 9-1. 흐름 점검 — 간과되기 쉬운 로직 & 처리 방침

개발 시 아래 엣지 케이스를 반드시 처리한다.

- **첫 실행/온보딩**: 로그인 후 프로필이 없으면 → 아이 프로필 생성 유도.
  프로필 없이 학습/테스트 진입 불가.
- **PIN 최초 설정 & 분실**: 부모 모드 첫 진입 시 PIN 설정. 분실 시 로그인된
  이메일 계정이므로 **재인증(로그아웃 후 재로그인) 기반 PIN 재설정** 제공.
- **활성 프로필 유지**: 현재 선택된 아이 프로필을 기기(localStorage)에 저장,
  앱 재실행 시 복원. 여러 프로필이면 상단에서 전환.
- **단어 부족 시 게임/테스트**:
  - 객관식 보기(4개)를 못 만들 만큼 단어가 적으면 → 보기 수 축소 또는 게임 잠금.
  - 그림 필요 게임인데 image/emoji 단어가 부족 → 게임 목록에서 숨김.
  - 테스트 문항 수 > 등록 단어 → 있는 만큼만 출제.
- **단어 0개**: 아이 홈은 "아직 단어가 없어요 🐣" 빈 상태 + 부모 등록 안내.
- **모든 단어 mastered**: 학습할 새 단어가 없으면 복습 세션(가끔 재노출)으로 전환.
- **채점 일관성**: 게임/테스트 모두 오답 시 `wrong_count`+1 & `streak=0`,
  정답 시 `streak`+1(3회 → mastered)로 동일하게 처리.
- **TTS/음성인식 미지원**: 브라우저에 en 음성이 없거나 SpeechRecognition 미지원 시
  해당 기능(발음 게임 등)을 숨기고 텍스트/버튼으로 대체(feature detection).
- **프로필 삭제**: 해당 프로필의 words/test_results도 함께 삭제(FK cascade).
- **중복 단어 입력**: 같은 프로필에 동일 `term` 추가 시 경고(허용은 하되 안내).

## 10. 결정 완료 & 열린 질문

### 결정 완료
- 단어 뜻: 대표 뜻(`meaning`) 채점 기준 + 추가 뜻(`alt_meanings`) 참고용.
- 이미지 어려운 단어: `image_url` 선택, `emoji` 대체, 그림 게임은 image/emoji 있는 단어만.
- 부모 모드 PIN 잠금 + `/progress` 자주 틀리는 단어 통계.
- `mastered` 승격 = 연속 정답 3회.
- 테스트 문항 수 5~10, **부모 모드에서 선택**(기본 10).
- 발음: 원어민 음성(en-US/en-GB) + 속도·톤 조절.
- 게임: 기본 4종 + 스펠링 빈칸 + 그림→단어 + 문장 빈칸 + 발음 따라 말하기.

### 열린 질문 (추후 확정)
- **TTS 품질**: MVP는 Web Speech API로 시작하되, 원어민 발음 일관성이 부족하면
  클라우드 TTS(예: Google/Amazon Polly, 비용·백엔드 발생)로 업그레이드할지?
- 이모지 외에 **기본 그림 세트/아이콘 라이브러리**도 제공할지?
- 테스트 기본 문항 수(예: 10문항?), 게임 세션 단어 수 기본값.
- 발음 인식(말하기 게임) 정확도가 낮을 때 대체 판정(예: "충분히 시도하면 통과") 기준.
